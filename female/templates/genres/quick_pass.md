# 快穿/穿书 Genre Template (女频)

## 概述

**类型**: 快穿 (Quick Pass) / 穿书 (Transmigration into Books)
**目标读者**: 喜欢系统驱动故事、多线叙事的女性读者
**核心元素**: 多个世界/系统、任务驱动情节、跨篇章关系发展

> **注意**: 快穿和穿书不同但经常结合：
> - **快穿 (Quick Pass)**: 主角穿越多个世界/系统
> - **穿书 (Transmigration)**: 主角进入她知道的小说/书籍

---

## 主要细分类型

### 类型一：攻略型 (Mission Romance)

| 元素 | 描述 |
|------|------|
| **主要目标** |完成任务获取积分/奖励 |
| **恋爱** | 在每个世界发展关系 |
| **重点** | 跨世界攻略恋爱 |
| **示例任务** | 攻略高冷男主/治愈黑化男配/攻略深情男二 |

**故事弧线**:
1. 进入世界 → 遇见爱人 → 完成 tasks 赢得爱情 → 世界结束 → 进入下一个

### 类型二：收集型 (Collection)

| 元素 | 描述 |
|------|------|
| **主要目标** | 收集物品/能力/记忆 |
| **恋爱** | 跨世界有多位伴侣 |
| **重点** | 边冒险边积累实力 |

### 类型三：逆袭型 (Counterattack)

| 元素 | 描述 |
|------|------|
| **原故事** | 有受害者的主角 |
| **主角角色** | 进入并改变命运 |
| **重点** | 复仇、洗清冤屈、幸福结局 |
| **常见模式** | 原本的女主是受害者 |

**故事弧线**:
1. 确定原始悲剧 → 改变关键事件 → 澄清误会 → 为受害者带来幸福结局

### 类型四：治愈型 (Healing)

| 元素 | 描述 |
|------|------|
| **原故事** | 有悲剧角色 |
| **主角角色** | 进入治愈破碎的人 |
| **重点** | 情感治愈、救赎 |
| **基调** | 温暖、情感 |

### 类型五：事业型 (Career Focus)

| 元素 | 描述 |
|------|------|
| **重点** | 跨世界的职业成功 |
| **恋爱** | 次要或最少 |
| **任务** | 商业/职业相关 |

### 类型六：命运纠葛型 (Fate Entanglement)

| 元素 | 描述 |
|------|------|
| **模式** | 原角色跨世界记得 |
| **恋爱** | 主角形成深厚羁绊 |
| **冲突** | 系统冲突 vs 情感 |
| **解决** | 跨世界解决 |

**故事弧线**:
1. 进入世界 → 遇见爱人 → 完成 tasks 赢得爱情 → 世界结束 → 进入下一个

## 系统设计框架

### 用户定义元素 (USER-DEFINED ELEMENTS)

| 元素 | 问题 |
|------|------|
| **任务目标** | 主角必须完成什么？ |
| **成功标准** | 如何衡量完成？ |
| **奖励** | 主角获得什么？ |
| **后果** | 失败时会发生什么？ |

### 常见系统机制

```
系统类型：任务系统/积分系统/好感度系统/生命值系统
任务类型：攻略/收集/拯救/反攻略/破坏
难度等级：简单/中等/困难/地狱
奖惩机制：奖励丰厚/惩罚严重/失败无惩罚
```

### 系统类型示例

| 类型 | 描述 | 示例 |
|------|------|------|
| **任务系统** | 完成特定目标 | 好感度达到100/收集5个道具 |
| **积分系统** | 跨世界积累积分 | 1000积分可兑换一个愿望 |
| **好感度系统** | 追踪关系等级 | 好感度0-100，需达到80 |
| **生命值系统** | 基于生命/生命值的进度 | 10条命，用完即死 |
| **混合系统** | 结合多种机制 | 任务+积分+好感度 |

### 任务类型

| 类型 | 描述 |
|------|------|
| **攻略** | 赢得某人的心 |
| **收集** | 收集物品/记忆/能力 |
| **拯救** | 从悲剧中拯救角色 |
| **反攻略** | 破坏或拒绝恋爱追求 |
| **破坏** | 打破现有关系/情节 |
| **治愈** | 治愈情感创伤 |
| **事业** | 实现职业目标 |

### 难度等级

| 等级 | 描述 |
|------|------|
| **简单** | 直接任务，成功率高 |
| **中等** | 适度挑战，有些障碍 |
| **困难** | 复杂任务，重大障碍 |
| **地狱** | 极高难度，几乎没有容错 |

### 奖励/惩罚框架

| 类型 | 描述 |
|------|------|
| **奖励丰厚** | 慷慨奖励，练级优势 |
| **惩罚严重** | 失败后果严重 |
| **失败无惩罚** | 无惩罚，只是不奖励 |
| **平衡型** | 奖励与风险相等 |

## 世界结构

### 每个世界必须具备

1. **设定**: 故事发生的地方
2. **角色**: 关键NPC和可能的恋爱对象
3. **任务**: 必须完成什么
4. **冲突**: 要克服的障碍
5. **结束条件**: 主角何时可以离开

### 世界类型示例

| 类型 | 设定 | 常见任务 |
|------|------|----------|
| **校园世界** | 学校/大学 | 攻略学霸/改善人际关系/逆袭成女神 |
| **宫廷世界** | 宫廷阴谋 | 宫斗/攻略皇子/改写悲剧结局 |
| **职场世界** | 企业 | 升职加薪/攻略上司/商业博弈 |
| **星际世界** | 科幻 | 星球联盟/机甲战士/跨种族恋爱 |
| **武侠世界** | 武侠 | 江湖恩怨/攻略盟主/修炼武功 |
| **娱乐圈** | 娱乐 | 顶流明星/攻略影帝/名利场斗争 |
| **玄幻世界** | 奇幻 | 修仙成神/攻略师尊/神器收集 |
| **现代都市** | 现代城市 | 豪门恩怨/攻略总裁/独立自主 |
| **末日世界** | 后末日 | 生存冒险/攻略异能者/重建秩序 |
| **民国世界** | 民国 | 谍战风云/攻略少帅/家国情怀 |

### 世界构建检查清单

- [ ] 每个世界有独特的氛围
- [ ] 每个世界的角色令人难忘
- [ ] 任务与世界设定相关
- [ ] 冲突感觉自然
- [ ] 结束条件清晰且令人满意

## 情节模式

### 模式一：单向攻略 (One-way Romance)

```
进入世界 → 遇见爱人 → 完成 tasks 赢得爱情 → 世界结束 → 进入下一个
```

**特点**:
- 每个世界都是独立的恋爱
- 主角控制关系
- 每个世界有清晰的恋爱胜利

### 模式二：命运纠葛 (Fate Entanglement)

```
原角色跨世界记得 → 形成深厚羁绊 → 系统 vs 情感冲突 → 跨世界解决
```

**特点**:
- 跨世界有连续线索
- 情感赌注随时间增加
- 系统规则 vs 个人感受冲突

### 模式三：逆袭翻盘 (Counterattack)

```
确定原始悲剧 → 改变关键事件 → 澄清误会 → 为受害者带来幸福结局
```

**特点**:
- 重点是正义和救赎
- 主角作为改变者
- 为原始受害者带来满意回报

### 模式四：收集养成 (Collection Cultivation)

```
进入世界 → 确定收集目标 → 收集资源 → 建立实力 → 更强地离开
```

**特点**:
- 实力进阶重点
- 各种收集目标
- 资源管理

### 模式五：治愈救赎 (Healing Redemption)

```
进入世界 → 确定受伤角色 → 建立信任 → 治愈情感伤害 → 和平离开
```

**特点**:
- 情感深度重点
- 温暖、令人满意的弧线
- 角色转变

## 数据库 Schema (SQL)

### Quick Pass Worlds Table

```sql
CREATE TABLE quick_pass_worlds (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    novel_id INTEGER NOT NULL,
    world_name VARCHAR(100) NOT NULL,
    world_type VARCHAR(50) NOT NULL,
    world_order INTEGER NOT NULL,
    missions TEXT NOT NULL,
    mission_type VARCHAR(50),
    completion_status VARCHAR(20) DEFAULT '进行中', -- 进行中/已完成/失败
    protagonist_status VARCHAR(100),
    love_interest VARCHAR(100),
    love_interest_archetype VARCHAR(50),
    romance_progress VARCHAR(20) DEFAULT '未开始', -- 未开始/暧昧中/已攻略/失败
    key_conflicts TEXT,
    end_condition VARCHAR(200),
    rewards TEXT,
    world_notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (novel_id) REFERENCES novels(id)
);
```

### Novel Main Table Extension

```sql
CREATE TABLE novels (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title VARCHAR(200) NOT NULL,
    genre VARCHAR(50) DEFAULT '快穿/穿书',
    sub_type VARCHAR(50),
    system_type VARCHAR(50),
    mission_type VARCHAR(50),
    difficulty_level VARCHAR(20),
    romance_carry_over BOOLEAN DEFAULT FALSE,
    character_carry_over BOOLEAN DEFAULT FALSE,
    total_worlds INTEGER,
    current_world INTEGER DEFAULT 1,
    protagonist_name VARCHAR(100),
    protagonist_personality TEXT,
    system_entity_description TEXT,
    user_defined_tasks TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### Character Relationships Table

```sql
CREATE TABLE quick_pass_characters (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    world_id INTEGER NOT NULL,
    character_name VARCHAR(100) NOT NULL,
    character_role VARCHAR(50), -- 男主/男配/女主/女配/反派/路人
    archetype VARCHAR(50),
    is_main_love_interest BOOLEAN DEFAULT FALSE,
    personality_traits TEXT,
    background_story TEXT,
    relationship_with_protagonist VARCHAR(50), -- 陌生/相识/暧昧/恋人/仇人
    romance_progress VARCHAR(20), -- 好感度数值或状态
    key_events TEXT,
    ending_status VARCHAR(20), -- 幸福/悲剧/开放/独美
    FOREIGN KEY (world_id) REFERENCES quick_pass_worlds(id)
);
```

### Mission Progress Table

```sql
CREATE TABLE mission_progress (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    world_id INTEGER NOT NULL,
    mission_name VARCHAR(200) NOT NULL,
    mission_description TEXT,
    mission_status VARCHAR(20) DEFAULT '未完成', -- 未完成/进行中/已完成/失败
    completion_criteria TEXT,
    current_progress VARCHAR(100),
    rewards TEXT,
    hints TEXT,
    FOREIGN KEY (world_id) REFERENCES quick_pass_worlds(id)
);
```

## 用户确认问题

### 创建任何快穿/穿书小说前

**必须与用户确认**：

| # | 问题 | 选项 |
|---|------|------|
| 1 | **任务类型**是什么？ | 攻略/收集/拯救/逆袭/治愈/混合 |
| 2 | **系统类型**是什么？ | 任务系统/积分系统/好感度系统/混合 |
| 3 | 恋爱是**主要还是次要**？ | 主导/辅助/可有可无 |
| 4 | 是否有**多位恋爱对象**？ | 每世界一个/固定一人/视情况 |
| 5 | 角色是否**跨世界携带**？ | 是/否/部分角色 |
| 6 | **难度等级**是什么？ | 简单/中等/困难/地狱 |
| 7 | 计划多少个**世界**？ | 具体数量或范围 |
| 8 | 是否有贯穿各世界的**主情节**？ | 有/无/有但弱 |

### 可选确认

| # | 问题 | 选项 |
|---|------|------|
| 9 | **节奏风格**是什么？ | 快节奏/中等/慢热 |
| 10 | 是否有**禁止元素**？ | 指定需避免的内容 |
| 11 | **结局偏好**？ | 单结局/多结局/开放结局 |
| 12 | **反嫉妒条款**？ | 女主独美/有其他追求者/暧昧向 |

## 世界间过渡技术

### 常见过渡

| 技术 | 描述 |
|------|------|
| **任务完成** | 当前任务结束，传送门打开 |
| **强制传送** | 系统强制过渡 |
| **生命耗尽** | 生命/生命值系统触发离开 |
| **主动选择** | 主角选择离开 |
| **触发结局** | 特殊条件提前结束世界 |
| **时间到期** | 现实世界截止日期强制离开 |

### 过渡元素

- 离开当前世界的情感时刻
- 新世界介绍的钩子
- 积累进度的回顾
- 角色重聚（如携带）
- 系统界面时刻
