# 双主角设计指南 - 女频小说

本指南支持多种双主角配置：
- **双男主**（用于BL/耽美）
- **双女主**（用于GL/百合）
- **群像**（3+主角）

---

## 关系动态（用户偏好）

| 类型 | 描述 |
|------|------|
| **平等型** | 两个主角有相等的地位、目标和能力 |
| **互补型** | 一方主导，一方辅助，但辅助方有独立价值 |
| **不对等型** | 根据用户偏好，一方明显主导 |

---

## 1. 每个主角必须具备

- [ ] 独立的目标和困境
- [ ] 自己的成长弧线
- [ ] 鲜明的性格
- [ ] 在自己领域的能力

---

## 2. 关系张力来源

| 来源 | 类型 | 描述 |
|------|------|------|
| 互补特质 | 冲突型 | 对立的优点创造摩擦 |
| 不同价值观 | 摩擦型 | 信念/世界观的冲突 |
| 共同目标 | 合作型 | 尽管有差异仍结盟 |
| 隐藏秘密 | 悬念型 | 未知过往或意图 |

---

## 3. 关系类型

### 双男主（耽美）

| 方面 | 指南 |
|------|------|
| **关系** | 攻方 vs 受方——两者都必须是完整的人 |
| **动态** | 谁追求谁，权力平衡 |
| **发展** | 从陌生→羁绊→深度绑定 |
| **核心原则** | 攻受不等于强弱；两者都有独立生活 |

### 双女主（百合）

| 方面 | 指南 |
|------|------|
| **关系** | 两者都是积极参与者 |
| **动态** | 竞争/合作/复杂 |
| **发展** | 从认识→吸引→羁绊 |
| **核心原则** | 两者都有独立的事业和目标 |

### 群像

| 方面 | 指南 |
|------|------|
| **核心三角** | 最重要的3个角色 |
| **多视角** | 每个都有自己的弧线 |
| **平衡** | 没有单一主角掩盖其他角色 |
| **焦点轮换** | 谁承担当前弧线 |

---

## 4. 叙事策略——混合模式

混合模式结合日常交替与重要节点轮换。

### 日常交替

- 日常场景：每几段或每个场景交替
- 示例：第5章主角A的日常生活→第6章主角B的日常生活

### 重要节点轮换

- 关键事件：轮换焦点或使用双视角
- 高潮场景：同一事件的两方视角
- 示例：第10章（高潮）：A和B对抗的双视角

---

## 5. 章节结构示例

### 示例1——双男主耽美

```
第X章：意外相遇

[攻方视角 - 3段]
[受方视角 - 3段]
[攻方视角 - 2段]
[受方视角 - 2段]

→ 汇聚于同一时刻
```

### 示例2——双女主百合

```
第X章：职场交锋

[女主A视角 - 全章]
→ 以悬念结尾

第X+1章：[女主B视角 - 全章]
→ 同一事件的不同角度
```

### 示例3——群像

```
第X章：关键抉择

[A视角 - A的决策过程]
[B视角 - B的反应]
[C视角 - C的反击]

→ 汇聚
```

## 6. 关系发展追踪

```sql
CREATE TABLE dual_relationships (
    id INTEGER PRIMARY KEY,
    protagonist_a_id INTEGER NOT NULL,
    protagonist_b_id INTEGER NOT NULL,
    relationship_stage VARCHAR(50)
        CHECK (relationship_stage IN (
            '初识', '暧昧', '确认', '深化', '危机', '升华'
        )),
    tension_level INTEGER CHECK (tension_level BETWEEN 1 AND 10),
    equality_level VARCHAR(20)
        CHECK (equality_level IN ('equal', 'complementary', 'unequal')),
    conflict_points TEXT,
    UNIQUE(protagonist_a_id, protagonist_b_id)
);
```

## 7. 设计检查点

- [ ] 两者都有独立目标？
- [ ] 两者都有鲜明性格？
- [ ] 关系张力有意义？
- [ ] 两者都不是对方的工具？
- [ ] 用户偏好的关系风格保持一致？
- [ ] 叙事交替流畅？
- [ ] 双视角增强而非重复？

---

## 8. 用户偏好问题（规划阶段）

开始前需澄清：

| 问题 | 选项 |
|------|------|
| 关系风格 | 平等/互补/不对等 |
| 叙事焦点 | 哪方主角在前几章主导？ |
| 敏感内容 | 如何处理亲密场景？ |
| 主角配置 | 双男主(BL)/双女主(GL)/群像(Group) |

## 快速参考卡

| 配置 | 视角模式 | 关键张力 | 发展弧线 |
|------|----------|----------|----------|
| 双男主 | 交替，关键场景汇聚 | 攻受动态+个人目标 | 陌生→羁绊→深度绑定 |
| 双女主 | 每章或交替 | 竞争/合作+独立事业 | 认识→吸引→羁绊 |
| 群像 | 轮换多视角 | 三角核心+个人弧线 | 因角色而异 |
