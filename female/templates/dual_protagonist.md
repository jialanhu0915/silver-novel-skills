# 双主角设计指南 - 女频小说

本指南支持多种双主角配置：
- **双男主**（用于BL/耽美）
- **双女主**（用于GL/百合）
- **群像**（3+主角）

---

## 关系动态（用户偏好）

| 类型 | 描述 |
|------|------|
| **平等型** | 两个主角有相等的地位、目标和能力 |
| **互补型** | 一方主导，一方辅助，但辅助方有独立价值 |
| **不对等型** | 根据用户偏好，一方明显主导 |

---

## 1. 每个主角必须具备

- [ ] 独立的目标和困境
- [ ] 自己的成长弧线
- [ ] 鲜明的性格
- [ ] 在自己领域的能力

---

## 2. 关系张力来源

| 来源 | 类型 | 描述 |
|------|------|------|
| 互补特质 | 冲突型 | 对立的优点创造摩擦 |
| 不同价值观 | 摩擦型 | 信念/世界观的冲突 |
| 共同目标 | 合作型 | 尽管有差异仍结盟 |
| 隐藏秘密 | 悬念型 | 未知过往或意图 |

---

## 3. 关系类型

### 双男主（耽美）

| 方面 | 指南 |
|------|------|
| **关系** | 攻方 vs 受方——两者都必须是完整的人 |
| **动态** | 谁追求谁，权力平衡 |
| **发展** | 从陌生→羁绊→深度绑定 |
| **核心原则** | 攻受不等于强弱；两者都有独立生活 |

**场景演示：初遇——同一时刻，两种目光**

> 背景：楚砚（外科医生）和沈霁（独立音乐人），凌晨一点半的便利店，第一次相遇。

**❌ 无差异视角：** 楚砚看到黑衣人和伤口→沈霁也看到白大褂和手→两人内心活动完全一致（"他看起来很累"），换掉名字彼此通用。

**✅ 职业滤镜下的双视角：**

**【楚砚视角】**

他推开便利店的门，手指还在微微发抖——那台手术缝了四个小时血管。柜台边站着一个穿黑卫衣的人，帽子拉得很低。楚砚扫了一眼那人的左手：虎口有一道没结痂的伤口，指甲掐的。脑海中闪过诊断：焦虑，习惯性抠手指。

**【沈霁视角】**

沈霁睁开眼。一罐黑咖啡放在柜台上，磕碰声把他从demo的和弦进行里拽出来。拿咖啡的人穿着白大褂，口袋里夹着一支没盖好的笔。消毒水的味道。沈霁注意到他的手指——食指指腹有茧，不是弹琴的，是握刀柄的。外科医生。

他忽然把自己的手插进了口袋。虎口上那道自己掐出来的月牙，还在隐隐发疼。

**【楚砚视角】**

楚砚走出店门，发现又下雨了。屋檐下，他拧开咖啡罐。

"拿去。"

那个穿黑卫衣的人站在门口，手里拎着一把伞。没看他。
"你手指在发抖。再站一会儿会感冒的。"

楚砚接过伞。那人转身回去结矿泉水了。

**【沈霁视角】**

他把矿泉水递给收银员，从玻璃里看到那个医生撑着伞走远了。伞很小，但那个人撑着，刚好。

他忽然觉得今晚的歌知道往哪走了——不是金属，是雨。

**拆解：**
- 同一双手，两种职业滤镜：医生看到"焦虑、抠手指"，音乐人看到"握刀柄的茧子"——视角延展角色
- 对话仅三句，两个不善言辞的人，用动作和观察替代长篇对白
- 结尾回到各自内心：沈霁的创作线索"是雨"呼应开头的demo，双视角在一条暗线交汇

### 双女主（百合）

| 方面 | 指南 |
|------|------|
| **关系** | 两者都是积极参与者 |
| **动态** | 竞争/合作/复杂 |
| **发展** | 从认识→吸引→羁绊 |
| **核心原则** | 两者都有独立的事业和目标 |

**场景演示：同一场会议，两个世界**

> 背景：季晚（建筑师）和洛时雨（景观设计师）在竞标会上竞争同一个项目。季晚在台上汇报方案。

**❌ 重复视角：** 季晚看到"洛时雨在看我，好像在挑刺"→洛时雨看到"季晚方案不错，我有点嫉妒"→两人的内心独白是同一件事的两个版本，没有信息增量。

**✅ 信息互补的双视角：**

**【季晚视角】**

她翻到最后一页效果图，手心在出汗。台下甲方代表面无表情，洛时雨双臂交叉坐在最后排——季晚认识这个表情，是来找茬的。

"季工，"洛时雨举手，"山体渗水你怎么解决？"

会议室安静了。季晚看着洛时雨，忽然明白了什么。她翻到防洪剖面图："时雨的问题很好。我们在表面做了三层隔水——渗水不是问题，是资源，能供应60%日常用水。"

洛时雨点了一下头。不是"算你过关"——是"你确实准备好了"。

那个打断不是找茬，是把她推到甲方最关心的问题上。

**【洛时雨视角】**

季晚的方案确实好。好到她有点不甘心。但季晚的手指在遥控器边缘不安地摩挲着——洛时雨想起上周在地下车库，季晚发动了三遍车都打不着火。这个人，越紧张越不说话。

她举起手打断了她。问了一个甲方还在犹豫的问题。

散会后电梯口，季晚追上来："我还以为你是来找茬的。"
洛时雨按着电梯门："我是来找茬的，顺便帮个小忙。下个月我的方案汇报——你来不来？"
季晚看着电梯数字："来。我会先想好问题再来。"

**拆解：**
- 同一事件两面信息：季晚看到"挑衅"→发现"帮助"；洛时雨清楚自己在帮忙
- 每个视角只保留角色的感官边界：季晚不知道洛时雨之前想了什么，洛时雨不知道季晚什么时候发现
- 双视角放大了关系动态：读者同时看到两位各自的小心思

### 群像

| 方面 | 指南 |
|------|------|
| **核心三角** | 最重要的3个角色 |
| **多视角** | 每个都有自己的弧线 |
| **平衡** | 没有单一主角掩盖其他角色 |
| **焦点轮换** | 谁承担当前弧线 |

## 4. 叙事策略——混合模式

混合模式结合日常交替与重要节点轮换。

### 日常交替

- 日常场景：每几段或每个场景交替
- 示例：第5章主角A的日常生活→第6章主角B的日常生活

### 重要节点轮换

- 关键事件：轮换焦点或使用双视角
- 高潮场景：同一事件的两方视角
- 示例：第10章（高潮）：A和B对抗的双视角

---

## 5. 章节结构示例

### 示例1——双男主耽美

```
第X章：意外相遇

[攻方视角 - 3段]
[受方视角 - 3段]
[攻方视角 - 2段]
[受方视角 - 2段]

→ 汇聚于同一时刻
```

### 示例2——双女主百合

```
第X章：职场交锋

[女主A视角 - 全章]
→ 以悬念结尾

第X+1章：[女主B视角 - 全章]
→ 同一事件的不同角度
```

### 示例3——群像

```
第X章：关键抉择

[A视角 - A的决策过程]
[B视角 - B的反应]
[C视角 - C的反击]

→ 汇聚
```

## 6. 关系发展追踪

```sql
CREATE TABLE dual_relationships (
    id INTEGER PRIMARY KEY,
    protagonist_a_id INTEGER NOT NULL,
    protagonist_b_id INTEGER NOT NULL,
    relationship_stage VARCHAR(50)
        CHECK (relationship_stage IN (
            '初识', '暧昧', '确认', '深化', '危机', '升华'
        )),
    tension_level INTEGER CHECK (tension_level BETWEEN 1 AND 10),
    equality_level VARCHAR(20)
        CHECK (equality_level IN ('equal', 'complementary', 'unequal')),
    conflict_points TEXT,
    UNIQUE(protagonist_a_id, protagonist_b_id)
);
```

## 7. 设计检查点

- [ ] 两者都有独立目标？
- [ ] 两者都有鲜明性格？
- [ ] 关系张力有意义？
- [ ] 两者都不是对方的工具？
- [ ] 用户偏好的关系风格保持一致？
- [ ] 叙事交替流畅？
- [ ] 双视角增强而非重复？

---

## 8. 用户偏好问题（规划阶段）

开始前需澄清：

| 问题 | 选项 |
|------|------|
| 关系风格 | 平等/互补/不对等 |
| 叙事焦点 | 哪方主角在前几章主导？ |
| 敏感内容 | 如何处理亲密场景？ |
| 主角配置 | 双男主(BL)/双女主(GL)/群像(Group) |

## 快速参考卡

| 配置 | 视角模式 | 关键张力 | 发展弧线 |
|------|----------|----------|----------|
| 双男主 | 交替，关键场景汇聚 | 攻受动态+个人目标 | 陌生→羁绊→深度绑定 |
