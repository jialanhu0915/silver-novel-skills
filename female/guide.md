# 女频小说创作指南

> 本指南是 `silver-novel-skills` Skill 的女频子流程，由根 `SKILL.md` 路由后通过 Read 工具加载。
> 不要直接作为独立 Skill 使用（无 frontmatter，不会被 Claude Code 自动发现）。

---

专为女频读者打造：言情、耽美、百合、女尊、快穿、悬疑恋爱等。核心是「感情线推进」+「男性角色独立性」+「女性主角深度塑造」。

---

## 工作流程（核心）

### 阶段一·规划

**目标：** 完成小说基本框架（类型、感情线风格、主角模式 → 角色、世界观、情节、金手指）。

**AI 动作：**

1. 收集用户偏好（一次最多 3 个问题）：类型 / 感情线风格（甜宠/虐恋/推拉）/ 角色模式（平等/互补/主从/双男主BL/双女主GL/群像）
2. Read `female/prompts/planning_flow.md`，按其步骤推进
3. 在以下节点按需 Read INDEX 选择文件（**一次只加载 1 个 Agent**）：
   - **题材模板：** Read `female/templates/genres/INDEX.md` → 选择 1 个对应题材
   - **框架构建 Agent：** Read `shared/agents/framework/INDEX.md` → 选择（世界/角色/情节）
   - **女频专项模板：** Read `female/templates/INDEX.md` → 按需选择（女主/双主角/感情线/金手指/反派打脸）
   - **角色关系网络：** Read `shared/templates/INDEX.md` → 按需选择
4. 提供 A/B 选项：用户补充细节 vs AI 自主设定
5. ⏸ 等待用户确认整体框架

**进入条件：** 用户选择女频 / 用户尚未提供完整大纲
**退出条件：** 用户确认「框架 OK，进入创作」

---

### 阶段二·逐章创作

**目标：** 根据框架逐章创作，感情线推进优先，数据库全程追踪设定一致性。

**AI 动作：**

1. Read `female/prompts/female_chapter_flow.md`，按其步骤推进（重点：感情线推进、心理描写、微表情）
2. **写作技法：** Read `shared/templates/writing_craft/INDEX.md` → 按当前章节焦点选择 1 个（情感/对白/场景/视角/世界观/深度平衡）
3. 每章创作前查询数据库（最近 10 章概要、角色状态、感情线阶段、待回收伏笔）
4. 每章完成后入库：章节内容、角色事件、感情线进展、伏笔更新
5. ⏸ 每章完成后等待用户审批

**进入条件：** 用户确认框架完成
**退出条件：** 用户要求停止 / 篇幅目标达成

---

### 阶段三·评测

**目标：** 对章节进行多维度质量检查并记录问题（特别关注男性角色独立性与感情线质量）。

**AI 动作：**

1. Read `female/prompts/review_flow.md`，按其顺序调用评测 Agent
2. **通用评测 Agent：** Read `shared/agents/review/INDEX.md` → 根据章节问题选择评测维度（13 个跨题材）
3. **女频专项评测（重点）：** Read `female/agents/review/INDEX.md` → 选择 1 个或多个（4 个女频专项，`male_roles_judge` 为 Critical）
4. 收集反馈：记录到 `feedbacks` 表
5. ⏸ 等待用户对评测结果的反馈（接受 / 决定重写）

**进入条件：** 章节入库后 / 用户主动要求评测
**退出条件：** 用户接受结果 / 决定重写章节

## 核心设计原则（必读）

### 一、智慧型反派

**反派的嘲讽必须有智慧、有根据，禁止无脑降智！**

| 检查项 | 说明 |
|--------|------|
| 嘲讽有事实依据 | 基于客观事实或合理推断 |
| 嘲讽有目的 | 试探/施压/立威/离间 |
| 反派有城府 | 不只靠嘴，有实际手段 |
| 嘲讽分对象 | 对强者有应有的尊重 |

### 二、男性角色独立人格（关键问题）

**男性角色不是「帅/富/专一」的工具人，他们是自己人生的主角。**

| 检查项 | 说明 |
|--------|------|
| 有独立目标 | 不是只为「爱女主」而活 |
| 有自己成长线 | 不是被女主「改造」 |
| 有自己困境 | 需要解决自己的问题 |
| 有底线原则 | 不是无条件的付出 |
| 有独立人生 | 工作/理想/社交不围绕女主转 |

### 三、女性主角深度塑造

**女主不是等待被拯救的公主，是自己故事的主角。**

| 检查项 | 说明 |
|--------|------|
| 有情感独立性 | 不是离了男人不能活 |
| 有智慧与判断 | 能独立分析问题 |
| 有成长弧光 | 性格会变化，有心路历程 |
| 有自己的选择 | 她的决定推动剧情 |
| 有底线与原则 | 不是无条件的付出 |

### 四、感情线质量设计

**好的感情线是推拉、是张力、是甜虐交织。**

| 检查项 | 说明 |
|--------|------|
| 推拉张力 | 相爱相杀/欲擒故纵/误会虐点 |
| 甜虐平衡 | 甜蜜与虐心交织，不过分甜腻或虐 |
| 情感递进 | 层层递进，有节奏的情感升温 |
| 心理细节 | 内心独白、微表情、行为暗示 |
| 关系平等 | 双向奔赴，非单方面付出 |

### 五、金手指使用原则

**金手指是辅助工具，不是替代成长的捷径。**

| 检查项 | 说明 |
|--------|------|
| 不是100%成功 | 有失败可能 |
| 有代价限制 | 消耗/次数/副作用 |
| 需要主角判断 | 不是自动触发 |
| 有依赖后果 | 过度依赖有负面影响 |

### 六、合理爽点设计

**爽点不只是「获得」，更是「成长」和「认同」。**

| 爽点类型 | 说明 |
|----------|------|
| 实力成长型 | 付出努力后的突破 |
| 情感共鸣型 | 信任回报、羁绊加深 |
| 认知升级型 | 看穿真相、智破困局 |
| 关系互动型 | 被认可、并肩作战 |
| 反派打脸型 | 智慧型打脸，有铺垫有手段 |

### 七、配角独立价值

**配角不是工具，是自己故事里的主角。**

| 检查项 | 说明 |
|--------|------|
| 有自己目标 | 不是只为帮助主角 |
| 有自己困境 | 需要解决自己的问题 |
| 有自己的关系 | 和主角是平等关系 |
| 有自己的成长 | 配角也在成长 |

## 用户风格偏好

进入阶段一时，向用户确认以下女频特色偏好：

### 1. 双主角关系风格

- **平等型**：双方实力/地位对等，势均力敌
- **互补型**：性格/能力互补，互相成就
- **主从型**：一方主导，另一方配合

### 2. 感情线风格

- **甜宠主**：甜蜜为主，虐点只是调味
- **虐恋主**：虐心为主，甜只是希望
- **推拉型**：甜虐交织，张力十足

### 3. 男性角色定位

- **并肩作战型**：与女主实力对等，共同战斗
- **守护型**：实力强但尊重女主选择
- **辅助型**：提供资源/情报，不抢风头

### 4. 女性被男性拯救

- **禁止型**：女主完全独立，不依赖男性拯救
- **有限型**：可以接受有限度的帮助
- **接受型**：可以接受英雄救美

## 感情线设计指南

### 推拉动力学

| 阶段 | 推 | 拉 |
|------|----|----|
| 暧昧期 | 故意疏远/假装不在意 | 暗中关心/不经意靠近 |
| 确认期 | 傲娇拒绝/欲擒故纵 | 真心流露/主动表白 |
| 磨合期 | 争吵冷战/误会 | 和好甜蜜/信任加深 |
| 稳定期 | 独立空间/小争执 | 默契十足/互相支持 |

### 甜虐平衡公式

```
总感情线 = 70%甜蜜片段 + 30%虐心片段

虐心类型分布：
- 误会虐：40%（可快速解开）
- 身份虐：30%（身世/立场的冲突）
- 信任虐：20%（背叛或隐瞒）
- 生死虐：10%（生死关头的考验）
```

### 心理描写技巧

- **微表情**：眼神/动作暗示情感
- **内心独白**：内心挣扎与抉择
- **行为逻辑**：行为反映心理变化
- **环境呼应**：场景烘托情感
- **对话张力**：话里有话/欲言又止

## 金手指设计（女频适配）

### 女频常见金手指类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 读心术 | 感知他人情绪/意图 | 谈判/社交无往不利 |
| 医术/毒术 | 治病救人/克敌制胜 | 神医流 |
| 空间/系统 | 储物/任务/技能兑换 | 快穿流 |
| 重生 | 先知优势 | 逆袭流 |
| 身份切换 | 多重马甲 | 马甲流 |
| 特殊体质 | 体质带来的优势 | 锦鲤体质/旺夫 |

### 金手指使用原则

1. **不为替代成长**：金手指是加速器，不是替代努力
2. **代价限制**：使用有代价，形成约束
3. **失败可能**：关键时刻可能失效
4. **依赖后果**：过度依赖会带来问题
5. **女性赋能**：金手指强化女性自主性
