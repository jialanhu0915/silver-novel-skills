# 配角设计指南

## 核心原则

**配角不是工具，他们是自己故事里的主角。**

好的配角：
- 有自己的目标和动机
- 有自己的问题需要解决
- 和主角是平等关系
- 存在价值不只是"帮助主角"或"被主角打败"

---

## 问题批判

### ❌ 配角沦为工具

```
❌ 工具人A：被主角救，然后永远感谢主角
❌ 工具人B：被主角打败，然后成为小弟
❌ 工具人C：只在需要时出现，不需要就消失
❌ 工具人D：全程打酱油，毫无存在感
```

**问题**：配角没有自己的生命。

### ❌ 配角脸谱化

```
❌ 老好人：永远帮助主角
❌ 墙头草：谁强跟谁
❌ 阴谋家：表面好实际坏
❌ 酱油党：除了喊"666"什么都不做
```

## 配角类型设计

### 1. 盟友型

| 要素 | 描述 |
|------|------|
| 和主角的关系 | 朋友/伙伴/队友 |
| 他们自己的问题 | 盟友有自己的困境 |
| 和主角的冲突点 | 盟友可能和主角有分歧 |
| 成长 | 盟友也在成长 |

**示例**：
```
❌ 错误写法：
盟友：主角的小弟，永远支持主角，永远在关键时刻帮忙

✅ 正确写法：
盟友A：有自己的目标，和主角方向一致所以合作
盟友B：和主角有竞争关系，但也有共同敌人
盟友C：曾经帮助过主角，但后来因为理念不合分道扬镳
```

### 2. 对手型（非反派）

| 要素 | 描述 |
|------|------|
| 和主角的关系 | 竞争/较量 |
| 他们自己的目标 | 对手有自己的目标 |
| 和主角的冲突点 | 竞争某样东西 |
| 可能的发展 | 对手可以变成盟友，也可以变成敌人 |

**示例**：
```
❌ 错误写法：
对手：纯粹为了衬托主角而被设计

✅ 正确写法：
对手A：想成为内门第一，主角挡了他的路
对手B：看不惯主角的作风，想教训他
对手C：和主角无冤无仇，但立场不同必须对立
```

### 3. 复杂型

| 要素 | 描述 |
|------|------|
| 和主角的关系 | 不是单纯的好或坏 |
| 他们自己的动机 | 他们的行为有自己合理的原因 |
| 和主角的关系变化 | 从敌对到理解/从合作到分歧 |
| 存在意义 | 让读者思考灰色地带 |

**示例**：
```
❌ 错误写法：
反派：坏的，所以我打败他是正确的

✅ 正确写法：
角色A：为了救自己家人不得不与主角为敌
角色B：做了坏事但有令人同情的理由
角色C：和主角站在对立面，但读者能理解他
```

### 4. 功能型（谨慎使用）

| 要素 | 描述 |
|------|------|
| 使用场景 | 只在特定情节需要时出现 |
| 设计要点 | 即使是功能型配角也要有自己的特点 |
| 避免 | 不要让功能型配角只有"工具"这一个特点 |

**示例**：
```
❌ 错误写法：
NPC：只负责提供任务，没有自己的故事

✅ 正确写法：
NPC：接受任务的商人，他有自己的生意经
NPC：提供情报的线人，他也有自己的敌人要对付
```

## 配角设计要素

### 每个重要配角都应该有

1. **自己的目标**
   - 他想要什么？
   - 这个目标和主角有关系吗？

2. **自己的问题/困境**
   - 他面临什么困难？
   - 主角能帮他吗？帮了会有什么后果？

3. **自己的性格**
   - 他不是"好人"或"坏人"，而是有自己性格的人
   - 他的性格如何影响他的选择？

4. **和主角的互动**
   - 不是单方面的"帮助主角"或"被主角打败"
   - 互动应该是双向的、有影响的

## 配角关系网络

### ❌ 错误关系网络

```
主角 ←→ 所有配角
（所有配角都直接和主角互动，没有自己的圈子）
```

### ✅ 正确关系网络

```
配角A ←→ 配角B（有自己的关系）
配角C ←→ 配角D（有自己的关系）
主角 ←→ 配角A（主角和A是朋友）
主角 ←→ 配角C（主角和C是对手）
配角A ←→ 配角C（配角A认识配角C，有自己的关系）
```

**每个配角不只和主角有关系，他们之间也有自己的关系网络。**

## 配角的戏份分配

### 主角vs配角的戏份平衡

| 类型 | 主角戏份 | 配角戏份 | 说明 |
|------|----------|----------|------|
| 单主角 | 70%+ | 30%- | 配角围绕主角 |
| 双主角 | 各40%+ | 20%- | 两个主要角色 |
| 群像 | 30%左右 | 每个配角有独立故事线 | 所有角色都有重要性 |

### 配角出场设计

```
✅ 合理安排：
配角A：在需要他的情节出现，有自己的故事
配角B：在特定阶段出现，推动剧情
配角C：偶尔出现，展示世界丰富度

❌ 不合理安排：
每个配角都在每个情节出现
配角的戏份超过他能承载的
配角出场后消失，再也不出现
```

---

## 配角与剧情设计

### 配角推动剧情的方式

```
✅ 正确方式：
配角A的选择 → 推动了剧情
配角B的问题 → 产生了冲突
配角C的成长 → 带来了变化

❌ 错误方式：
只有主角的选择能推动剧情
配角只是被动响应主角
```

## 配角检查清单

- [ ] 重要配角有自己的目标
- [ ] 重要配角有自己的困境/问题
- [ ] 配角之间的关系不只是"和主角的关系"
- [ ] 配角不会无缘无故帮助或反对主角
- [ ] 配角的行为有逻辑支撑
- [ ] 配角不会突然消失
- [ ] 配角的戏份和他能承载的故事匹配
- [ ] 没有纯粹的"工具人"设计
