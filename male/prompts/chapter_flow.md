# 章节创作流程

## 创作前准备

### 1. 查询相关设定
```sql
-- 获取小说基本信息
SELECT * FROM novels WHERE id = ?;

-- 获取角色设定
SELECT * FROM characters WHERE novel_id = ?;

-- 获取角色历史事件
SELECT ce.*, c.name
FROM character_events ce
JOIN characters c ON ce.character_id = c.id
WHERE c.novel_id = ?
ORDER BY ce.created_at;

-- 获取势力状态
SELECT * FROM factions WHERE novel_id = ?;

-- 获取物品状态
SELECT i.*, c.name as holder_name
FROM items i
LEFT JOIN characters c ON i.current_holder = c.id
WHERE i.novel_id = ?;

-- 获取章节大纲
SELECT * FROM frameworks
WHERE novel_id = ? AND type LIKE '%plot%'
ORDER BY id;

-- 获取已埋伏笔
SELECT * FROM frameworks
WHERE novel_id = ? AND category = 'foreshadow';
```

### 2. 确认本章要点
- 本章涉及的角色
- 本章涉及的事件
- 本章需要的伏笔
- 需要在本章回收的伏笔
- 与前文的呼应点

## 创作要求

### 基础要求
1. **保持设定一致性**：严格遵循已确立的设定
2. **人物行为一致**：角色行为符合其性格和背景
3. **推进情节**：本章需要有明确的情节推进
4. **埋设伏笔**：适当埋下后续章节的伏笔
5. **文风统一**：保持与整体风格一致

### 反派塑造要求（重点）

**反派的嘲讽必须有智慧、有根据，禁止无脑降智！**

#### ✅ 允许的反派言行
- 嘲讽基于客观事实或合理推断
- 嘲讽有明确目的（试探/施压/立威/离间）
- 反派有城府，不只靠嘴
- 反派会利用规则、背景、资源等方式打压
- 反派对强者有应有的尊重

#### ❌ 禁止的反派言行
- 纯粹的人身攻击（"废物"、"穷鬼"、"垃圾"）
- 毫无根据的嘲讽
- 反派智商明显低于正常水平
- 对强者（如长老）同样无脑嘲讽
- 嘲讽只是为了被"打脸"，毫无意义

#### 反派嘲讽示例

```
# 场景：世家子弟vs寒门弟子

❌ 低端写法（禁止）：
"哈哈哈，看你那穷酸样，这种地方也是你该来的？"

✅ 高端写法：
他看了一眼主角手中的下品灵石，微微一笑。
"师弟，你可知道内门弟子与外门弟子的资源差距？
光是每月的丹药配额，就是十倍之别。
我不是看不起你，只是想说明一个现实。"
（陈述事实，而非无脑嘲讽）

# 场景：师兄"提醒"师弟

❌ 低端写法：
"就你还想进内门？做梦吧！"

✅ 高端写法：
"师弟，有件事我得提醒你。
这次选拔，评委不只是看修为。
你最近得罪了王家的人，而王家的长老正是评审之一。
我是为你好，提前告诉你。"
（看似好心，实则暗示：你完了）
```

## 章节入库

```
章节创作完成后：

1. **字数统计**：统计实际字数
2. **提取关键事件**：识别本章发生的重大事件
3. **识别涉及角色/势力/物品**：列出所有出现的实体
4. **更新数据库**：
   - 章节内容入库
   - 角色事件入库
   - 物品事件入库（如有）
   - 势力事件入库（如有）

5. **反派塑造记录**（新增）：
   - 记录本章反派的嘲讽方式
   - 评估反派是否有智慧
   - 如有"无脑嘲讽"倾向，需标记
```

## 反派塑造质量标记

这些字段已在 database/schema.sql 中定义：

```sql
-- chapters.antagonist_quality: intelligent/average/brainless
-- feedbacks.feedback_type 可选值：antagonist_stupid（反派无脑）
```

**使用说明：**
- 创作时记录反派嘲讽方式
- 评测时评估反派是否有智慧
- 如有"无脑嘲讽"倾向，标记为brainless
